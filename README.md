# N-gram MySQL

Выгрузка N-грамм напрямую из MySQL базы данных. Извлекает данные из таблиц товаров, категорий, брендов, тегов и блог-постов без необходимости парсинга веб-страниц.

## Возможности

- **Прямая выгрузка из MySQL** — работает напрямую с базой данных
- **Множественные таблицы** — товары, категории, бренды, теги, блог
- **N-граммы (1-6)** — генерация от униграмм до 6-грамм
- **Фильтрация стоп-слов** — автоматическая очистка от мусорных слов
- **Кастомные стоп-слова** — поддержка файла `stop.csv`
- **Раздельный экспорт** — сохранение каждого N-грамма в отдельный CSV
- **JSON экспорт** — полные данные с ID источников
- **Только названия** — режим выгрузки только `shop_product.name`

## Установка зависимостей

```bash
python3 -m venv venv
source venv/bin/activate
pip install mysql-connector-python nltk python-dotenv
```

## Подготовка

### Создайте файл `.env`

Параметры подключения к базе данных:

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=your_user
DB_PASSWORD=your_password
DB_NAME=your_database
```

### (Опционально) Создайте файл `stop.csv`

Кастомные стоп-слова для фильтрации (одно слово/фразу на строку):

```csv
слово
нежелательная фраза
```

## Использование

### Выгрузить все данные из всех таблиц

```bash
python ngram_mysql.py
```

### Выгрузить только названия товаров (shop_product.name)

```bash
python ngram_mysql.py -name
```

### Справка

```bash
python ngram_mysql.py --help
```

## Структура базы данных

Скрипт работает со следующими таблицами:

### `shop_product`

Основная таблица товаров.

```sql
SELECT id, name, summary, description
FROM shop_product
WHERE status != 0
```

Поля:
- `name` — название товара
- `summary` — краткое описание
- `description` — полное описание

### `blog_post`

Таблица блог-постов.

```sql
SELECT id, url, title, text
FROM blog_post
```

Поля:
- `title` — заголовок поста
- `text` — содержание поста

### `shop_category`

Таблица категорий.

```sql
SELECT id, name, description, seo_description
FROM shop_category
WHERE status != 0
```

Поля:
- `name` — название категории
- `description` — описание
- `seo_description` — SEO описание

### `shop_productbrands`

Таблица брендов.

```sql
SELECT id, name, description, seo_description, `h1`
FROM shop_productbrands
WHERE hidden != 1
```

Поля:
- `name` — название бренда
- `h1` — заголовок H1
- `description` — описание
- `seo_description` — SEO описание

### `shop_tageditor_tag`

Таблица тегов.

```sql
SELECT id, title, description, description_extra
FROM shop_tageditor_tag
```

Поля:
- `title` — название тега
- `description` — описание
- `description_extra` — дополнительное описание

## Стоп-слова

Автоматически загружаются:

1. **NLTK stopwords** (русский + английский)
2. **Дополнительные стоп-слова**:
   - местоимения: ваш, который, свой, этот, тот...
   - служебные: могу, может, можно, надо, нужно...
   - технические: сайт, страница, раздел, меню...

3. **Кастомные из `stop.csv`** (если файл существует)

## Адаптация под другую базу

Для работы с другой базой данных:

1. Измените SQL запросы в функциях `fetch_*()`
2. Обновите параметры подключения в `.env`
3. Адаптируйте обработку полей под вашу структуру

### Пример адаптации

```python
def fetch_products(connection: mysql.connector.MySQLConnection) -> List[Dict]:
    """Загружает данные из вашей таблицы."""
    query = """
        SELECT id, field1, field2
        FROM your_table
        WHERE condition = 1
    """

    cursor = connection.cursor(dictionary=True)
    cursor.execute(query)

    results = []
    for row in cursor:
        # Ваша логика обработки
        results.append({
            'id': f"table_{row['id']}",
            'text': ' '.join([row['field1'], row['field2']])
        })

    cursor.close()
    return results
```

## Сравнение с ngram_parser.py

| Характеристика | ngram_parser.py | ngram_mysql.py |
|---------------|-----------------|----------------|
| Источник данных | Веб-страницы (HTTP) | MySQL база |
| Скорость | Зависит от сети | Быстрее |
| Нагрузка | HTTP запросы | SQL запросы |
| ID источников | URL | ID записей |
| N-граммы | 1-3 | 1-6 |
| Резервирование | Да | Нет (быстро) |

## Режим `-name` (только названия)

Выгружает только поле `name` из таблицы `shop_product`:

```bash
python ngram_mysql.py -name
```

Полезно для:
- Анализа только названий товаров
- Быстрого получения ключевых слов
- Генерации тегов из названий

## Ошибки

### Ошибка подключения

```
❌ Ошибка подключения к MySQL: Access denied for user 'user'@'host'
```

**Решение:** Проверьте параметры в `.env` файле.

### Ошибка: DB_USER и DB_PASSWORD должны быть указаны

```
❌ DB_USER и DB_PASSWORD должны быть указаны в .env файле
```

**Решение:** Добавьте `DB_USER` и `DB_PASSWORD` в `.env`.

### Ошибка: таблица не найдена

```
❌ Ошибка при загрузке данных: Table 'database.shop_product' doesn't exist
```

**Решение:** Проверьте название таблицы и адаптируйте SQL запросы.
